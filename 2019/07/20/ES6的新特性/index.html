let var const

对于 let var const ，先说说三者的区别：

  1. （块级作用域） let 可以声明块级作用域的变量，而 var 不行
  2. （变量提升） let 不存在变量提升，只允许在 let 声明之后才能使用变量，而 var 存在变量提升
  3. （变量重复声明）let 不能重复声明一个变量，var 可以
  4. const 声明的是常量，不能修改，也就是不能修改指向那个值的地址
  5. const 在声明时必须赋值

let 和 var 一般推荐使用 let，因为用 var 可能会出现一些不明的错误，比如：

for (var i = 0; i < 10; i++) {
  setTimeOut(i => {
    console.log(i)
  }, 1000)
}
//上面的代码，会输出 10 个 10

或者：
if(true){
  console.log(a)
}else{
  var a=2;
  console.log('2')
}
//undifiend 2

const 声明的常量，其实是个地址，该地址指向一个变量的值，说的‘不能修改’，其实指的是不能修改地址，里面的值还是可以改变的。
比如：

const obj={
  name:'lxy';
}
obj.name='cyy'//不报错
obj={name:'cyy'}//报错

箭头函数

箭头函数是函数的一种简写形式
（）里面包含参数，接着一个箭头符号，后面大括号里是函数体
()=>{}
相比于 function 形式 ，箭头函数的优点：

  1. 更简洁

  2. 函数体只有一个 return 语句时，可以省略return关键字和花括号

  3. 没有自己的 this,只会继承上一层作用域的 this
    例如：

    function写法：
var name='The Window'
var obj={
   name:"The Object"
   getStudentName:function(){
       return function(){
         return this.name
       }
   }
}
console.log(obj.getStudentName()())//'The Window'

    第一层作用域的 this 为 obj,而第二层（闭包）里的 this 没有继承上一层作用域的 this,而是它自己本身的 this,所以是 window。
    其实可以用中间变量来保存第一层的 this 值，写法如下：

    var name='The Window'
var obj={
   name:"The Object"
   getStudentName:function(){
       var that=this

       return function(){
         return that.name
       }
   }
}
console.log(obj.getStudentName()())//'The Object'

    箭头函数写法：

    var name='The Window'
var obj={
   name:"The Object"
   getStudentName:()=>{
       return ()=>that.name
   }
}
console.log(obj.getStudentName()())//'The Object'

参数处理

参数默认值

//ES6 之前，当未传入参数时，text = 'default'；
function printText(text) {
  text = text || "default"
  console.log(text)
}

//ES6；
function printText(text = "default") {
  console.log(text)
}
